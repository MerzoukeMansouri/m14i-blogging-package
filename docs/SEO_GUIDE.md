## Search Engine Visibility

### Sitemap, Robots.txt, and RSS Feeds

Make your blog discoverable by search engines and feed readers with these **4 simple files**:

#### 1. Sitemap (`app/sitemap.ts`)

```typescript
import { generateNextJsSitemap } from 'm14i-blogging/server';
import { createServerSupabaseClient } from '@/lib/supabase-client';

export default async function sitemap() {
  const supabase = createServerSupabaseClient();

  return generateNextJsSitemap(supabase, {
    baseUrl: 'https://example.com',
    additionalUrls: [
      {
        url: 'https://example.com',
        lastModified: new Date(),
        changeFrequency: 'daily',
        priority: 1,
      },
      {
        url: 'https://example.com/about',
        changeFrequency: 'monthly',
        priority: 0.8,
      },
    ],
  });
}
```

**Result:** `/sitemap.xml` - Auto-generated from your blog posts

#### 2. Robots.txt (`app/robots.ts`)

```typescript
import { generateNextJsRobots } from 'm14i-blogging/server';

export default function robots() {
  return generateNextJsRobots({
    baseUrl: 'https://example.com',
    disallow: ['/admin', '/api', '/_next'],
  });
}
```

**Result:** `/robots.txt` - Tells search engines what to crawl

#### 3. RSS Feed (`app/rss.xml/route.ts`)

```typescript
import { generateRSSFeed } from 'm14i-blogging/server';
import { createServerSupabaseClient } from '@/lib/supabase-client';

export async function GET() {
  const supabase = createServerSupabaseClient();

  const rss = await generateRSSFeed(supabase, {
    baseUrl: 'https://example.com',
    title: 'My Blog',
    description: 'Latest posts from my blog',
    language: 'en',
  });

  return new Response(rss, {
    headers: { 'Content-Type': 'application/xml' },
  });
}
```

**Result:** `/rss.xml` - RSS feed for subscribers

#### 4. Submit to Search Engines

After deploying, submit your sitemap:

**Google Search Console:**
1. Go to [search.google.com/search-console](https://search.google.com/search-console)
2. Add your property
3. Submit: `https://your-site.com/sitemap.xml`

**Bing Webmaster:**
1. Go to [bing.com/webmasters](https://www.bing.com/webmasters)
2. Add your site
3. Submit sitemap

✅ **Done!** Your blog is now discoverable by search engines.

## Overview

The m14i-blogging package provides **automatic SEO optimization** for your blog posts, including:

- ✅ Meta tag generation (title, description, keywords)
- ✅ Open Graph tags for social media
- ✅ Twitter Card tags
- ✅ JSON-LD structured data (Schema.org)
- ✅ Reading time calculation
- ✅ Auto-generated excerpts
- ✅ SEO validation and scoring
- ✅ Smart defaults and auto-fill

All SEO fields are **optional** - the package intelligently auto-generates missing data from your content!

```bash
npm install m14i-blogging

pnpm add m14i-blogging

yarn add m14i-blogging
```

```tsx
// config/seo.ts
import type { SEOConfig } from 'm14i-blogging';

export const seoConfig: SEOConfig = {
  // Required
  siteUrl: 'https://yourblog.com',

    url: 'https://yourblog.com/about',
    image: 'https://yourblog.com/avatar.jpg',

      twitter: '@yourhandle',

      height: 60,

  },
  enableStructuredData: true,
  autoGenerateDefaults: true,
```

## SEO Configuration

The `SEOConfig` object controls how SEO metadata is generated.

### Required Fields

```tsx
{
  siteUrl: string;    // Base URL of your site (e.g., 'https://example.com')
  siteName: string;   // Name of your website
}
```

### Optional Fields

```tsx
{
  // Default author (used when post has no author)
  defaultAuthor?: {
    name: string;
    url?: string;
    image?: string;
    bio?: string;
    social?: {
      twitter?: string;
      linkedin?: string;
      github?: string;
      [key: string]: string | undefined;
    };
  };

  // Default Open Graph image
  defaultImage?: {
    url: string;
    width?: number;
    height?: number;
    alt?: string;
  } | string;

  // Twitter handle for site (@username)
  twitterSite?: string;

  // Default language code (e.g., 'en', 'en-US')
  defaultLanguage?: string;

  // Publisher information for structured data
  publisher?: {
    name: string;
    url?: string;
    logo?: ImageMetadata | string;
  };

  // Enable JSON-LD structured data generation
  enableStructuredData?: boolean;

  // Auto-generate missing SEO fields
  autoGenerateDefaults?: boolean;
}
```

## BlogPost SEO Fields

The `BlogPost` type has been extended with comprehensive SEO fields:

### Basic SEO Fields

```tsx
{
  // Required fields
  title: string;              // Page title
  slug: string;               // URL slug

  // Optional SEO fields
  excerpt?: string;           // Brief description (used for meta description)
  featuredImage?: string;     // Featured image URL

  // Author information
  author?: {
    name: string;
    url?: string;
    image?: string;
    bio?: string;
    social?: Record<string, string>;
  };

  // Categorization
  category?: string;          // Post category
  tags?: string[];            // Post tags/keywords

  // Dates
  publishedDate?: string;     // ISO 8601 format
  modifiedDate?: string;      // ISO 8601 format
  createdAt?: string;         // Fallback for publishedDate
  updatedAt?: string;         // Fallback for modifiedDate
}
```

### Advanced SEO Fields

```tsx
{
  // Core SEO metadata
  seo?: {
    description?: string;              // Meta description (150-160 chars)
    keywords?: string[];               // Keywords array
    canonicalUrl?: string;             // Canonical URL
    robots?: string;                   // 'index, follow' | 'noindex, follow' etc.
    language?: string;                 // Language code
    alternateLanguages?: Array<{       // Alternate language versions
      language: string;
      url: string;
    }>;
    additionalMeta?: Record<string, string>;  // Custom meta tags
  };

  // Open Graph metadata
  openGraph?: {
    title?: string;                    // OG title (defaults to post title)
    description?: string;              // OG description
    image?: ImageMetadata | string;    // OG image with metadata
    images?: Array<ImageMetadata | string>;
    type?: string;                     // 'article' | 'website' | 'blog'
    url?: string;                      // Canonical URL
    siteName?: string;                 // Site name
    locale?: string;                   // Locale (e.g., 'en_US')
    article?: {
      publishedTime?: string;
      modifiedTime?: string;
      authors?: string[];
      section?: string;
      tags?: string[];
    };
  };

  // Twitter Card metadata
  twitter?: {
    card?: 'summary' | 'summary_large_image' | 'app' | 'player';
    site?: string;                     // @username of website
    creator?: string;                  // @username of content creator
    title?: string;                    // Twitter title
    description?: string;              // Twitter description
    image?: ImageMetadata | string;    // Twitter image
    imageAlt?: string;                 // Twitter image alt text
  };
}
```

### Image Metadata Type

```tsx
interface ImageMetadata {
  url: string;        // Image URL (required)
  alt?: string;       // Alt text for accessibility
  width?: number;     // Width in pixels
  height?: number;    // Height in pixels
  type?: string;      // MIME type (e.g., 'image/jpeg')
}
```

```tsx
const analysis = analyzeContent(post);

// Returns:

  wordCount: number;           // Total word count
```

```tsx
import {

const minutes = calculateReadingTimeFromSections(post.sections);
```

## SEO Validation

Validate your blog posts and get actionable SEO recommendations:

### Get SEO Score

```tsx
import { getSEOScore } from 'm14i-blogging';

const result = getSEOScore(post, seoConfig);

// Returns:
{
  score: number;              // 0-100
  grade: 'excellent' | 'good' | 'fair' | 'poor';
  warnings: string[];         // Array of SEO warnings
}
```

**Grade Scale:**
- `excellent` - 90-100 points
- `good` - 70-89 points
- `fair` - 50-69 points
- `poor` - 0-49 points

### Validate SEO

```tsx
import { validateSEO } from 'm14i-blogging';

const warnings = validateSEO(post, seoConfig);
// Returns array of warning strings

// Example warnings:
[
  "Title is too long (75 chars). Keep it under 60 characters.",
  "Meta description is too short (95 chars). Aim for 120-160 characters.",
  "No H1 heading found in content.",
  "Content is short (250 words). Longer content (600+ words) typically ranks better."
]
```

### Display SEO Dashboard

```tsx
function SEODashboard({ post, config }) {
  const analysis = analyzeContent(post);
  const score = getSEOScore(post, config);

  return (
    <div>
      <h2>SEO Analysis</h2>
      <div>
        <strong>Word Count:</strong> {analysis.wordCount}
      </div>
      <div>
        <strong>Reading Time:</strong> {analysis.readingTime} min
      </div>
      <div>
        <strong>SEO Score:</strong> {score.score}/100 ({score.grade})
      </div>

      {score.warnings.length > 0 && (
        <div>
          <h3>Warnings:</h3>
          <ul>
            {score.warnings.map((warning, i) => (
              <li key={i}>{warning}</li>
            ))}
          </ul>
        </div>
      )}
    </div>
  );
}
```

## Best Practices

### 1. Title Optimization

**Guidelines:**
- Keep titles between 50-60 characters (max 70)
- Include primary keyword near the beginning
- Make it compelling and click-worthy
- Use numbers when applicable ("10 Tips...", "5 Ways...")

**Example:**
```tsx
✅ Good: "10 React SEO Best Practices for 2024"
❌ Bad: "This is a really long title about React SEO optimization techniques and strategies for improving search rankings in 2024"
```

### 2. Meta Description

**Guidelines:**
- Aim for 150-160 characters
- Include target keywords naturally
- Write compelling copy that encourages clicks
- Include a call-to-action

**Example:**
```tsx
✅ Good: "Master React SEO with these proven strategies. Learn meta tags, Open Graph, and structured data. Improve rankings in 2024!"
❌ Bad: "SEO tips"
```

### 3. Featured Images

**Guidelines:**
- Use 1200×630 pixels for optimal Open Graph display
- File size under 1MB for fast loading
- Use descriptive filenames (react-seo-guide.jpg)
- Always include alt text

```tsx
const post: BlogPost = {
  featuredImage: 'https://example.com/react-seo-guide.jpg',
  openGraph: {
    image: {
      url: 'https://example.com/react-seo-guide.jpg',
      width: 1200,
      height: 630,
      alt: 'React SEO best practices infographic',
      type: 'image/jpeg',
    },
  },
};
```

### 4. Content Quality

**Guidelines:**
- Aim for 600+ words minimum
- Use clear heading hierarchy (H1 → H2 → H3)
- Only one H1 per page
- Include images and multimedia
- Break text into readable paragraphs

### 5. Keywords and Tags

**Guidelines:**
- Use 3-8 relevant tags
- Focus on long-tail keywords
- Don't keyword stuff
- Be specific and descriptive

```tsx
✅ Good: ['React SEO', 'Next.js optimization', 'Meta tags', 'JSON-LD']
❌ Bad: ['SEO', 'web', 'dev', 'code', 'react', 'javascript', 'programming', 'tutorial', 'guide']
```

### 6. Author Attribution

**Guidelines:**
- Always include author information
- Link to author profile page
- Add social media handles
- Use consistent author data across posts

```tsx
const author: AuthorInfo = {
  name: 'Jane Developer',
  url: 'https://example.com/author/jane-developer',
  image: 'https://example.com/authors/jane.jpg',
  bio: 'Senior React Developer with 10+ years experience',
  social: {
    twitter: '@janedev',
    github: 'https://github.com/janedev',
    linkedin: 'https://linkedin.com/in/janedev',
  },
};
```

### 7. Dates

**Guidelines:**
- Always include published date
- Update modified date when editing
- Use ISO 8601 format
- Show dates to users

```tsx
const post: BlogPost = {
  publishedDate: '2024-01-15T10:00:00Z',
  modifiedDate: '2024-01-20T15:30:00Z',
};
```

### 8. Canonical URLs

**Guidelines:**
- Always set canonical URL
- Use absolute URLs
- Point to primary version of content
- Avoid duplicate content issues

```tsx
const post: BlogPost = {
  seo: {
    canonicalUrl: 'https://example.com/blog/react-seo-guide',
  },
};
```

## Advanced Usage

### Custom Meta Tags

Add custom meta tags beyond the defaults:

```tsx
const post: BlogPost = {
  seo: {
    additionalMeta: {
      'theme-color': '#B87333',
      'apple-mobile-web-app-capable': 'yes',
      'mobile-web-app-capable': 'yes',
    },
  },
};
```

### Multi-language Support

Configure alternate language versions:

```tsx
const post: BlogPost = {
  seo: {
    language: 'en',
    alternateLanguages: [
      { language: 'fr', url: 'https://example.com/fr/blog/post-slug' },
      { language: 'es', url: 'https://example.com/es/blog/post-slug' },
    ],
  },
};
```

### Custom Robots

Control search engine crawling:

```tsx
// Allow indexing and following
seo: { robots: 'index, follow' }

// Prevent indexing (for drafts, private content)
seo: { robots: 'noindex, follow' }

// Prevent following links
seo: { robots: 'index, nofollow' }

// Both
seo: { robots: 'noindex, nofollow' }
```

### Manual Meta Generation

Generate meta tags programmatically:

```tsx
import {
  generateHTMLMetaTags,
  generateMetaTagsHTML,
  toJSONLD,
} from 'm14i-blogging';

// Get meta tags as array of objects
const metaTags = generateHTMLMetaTags(post, seoConfig);

// Get meta tags as HTML string
const html = generateMetaTagsHTML(post, seoConfig);

// Get JSON-LD as string
const jsonld = toJSONLD(generateArticleSchema(post, seoConfig));
```

### React Helmet Integration

Use with react-helmet in non-Next.js apps:

```tsx

function BlogPage({ post }) {

```

### Programmatic Testing

```tsx
import { getSEOScore, validateSEO } from 'm14i-blogging';

// In your tests
describe('Blog Post SEO', () => {
  it('should have good SEO score', () => {
    const score = getSEOScore(post, seoConfig);
    expect(score.score).toBeGreaterThan(70);
    expect(score.grade).toBe('good');
  });

  it('should have no critical warnings', () => {
    const warnings = validateSEO(post, seoConfig);
    const critical = warnings.filter(w =>
      w.includes('No meta description') ||
      w.includes('No featured image')
    );
    expect(critical).toHaveLength(0);
  });
});
```

## Troubleshooting

### Issue: Meta tags not appearing

**Solution:** Ensure you're using the components correctly:

```tsx
// App Router: Use generateMetadata
export async function generateMetadata() {
  return generateBlogMetadata(post, seoConfig);
}

// Pages Router: Use BlogHead inside <Head>
<Head>
  <BlogHead post={post} config={seoConfig} />
</Head>
```

### Issue: Open Graph image not showing

**Solution:** Verify image requirements:
- Image must be publicly accessible (no authentication)
- Recommended size: 1200×630 pixels
- File size under 1MB
- Use absolute URLs (not relative)

### Issue: Low SEO score

**Solution:** Check validation warnings:

```tsx
const score = getSEOScore(post, seoConfig);
console.log(score.warnings);
// Address each warning
```

### Issue: Structured data errors

**Solution:** Validate with Google's Rich Results Test:
1. Generate JSON-LD: `toJSONLD(generateArticleSchema(post, seoConfig))`
2. Copy output
3. Paste into https://search.google.com/test/rich-results
4. Fix any reported errors
